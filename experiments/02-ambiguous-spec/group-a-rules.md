# Group A Rules: English Communication (Control)

You are part of a 5-agent team building a task queue library in Python.

## Communication

Communicate **naturally in English**. There are no special rules or formats.
Use GROUP_MESSAGE and AGENT_MESSAGE as you normally would.
Write messages however feels natural — prose, bullet points, whatever works.

## Your Task

Read `experiments/02-ambiguous-spec/task-description.md` for specifications.
All code goes in `experiments/02-ambiguous-spec/group-a-code/`.

**NOTE:** The spec is intentionally high-level. You will need to make design decisions as a team. The architect should clarify ambiguities. Ask questions if anything is unclear.

## Roles

- **Architect**: Design interfaces, clarify ambiguities, coordinate developers, resolve conflicts
- **Developer A**: Implement `models.py` + `__init__.py`
- **Developer B**: Implement `engine.py`
- **Reviewer**: Review all code, post feedback, approve when ready
- **Tester**: Write `test_queue.py`, run tests, report pass/fail

## Process

1. Architect reads the spec and designs the interfaces
2. Developers implement in parallel
3. Reviewer reviews code
4. Tester writes and runs tests
5. Fix any failures, re-test until all tests pass

## Rules

- Communicate as much or as little as you want
- No constraints on message format, length, or frequency
- Coordinate however feels natural
- Goal: all tests passing, clean code
